### 1. 基于序列化技术的病理的多实例学习
**时间**：2025.10 - 至今

项目合作单位：上海公利医院病理科
面向数字病理场景，参与基于序列化建模策略的多实例学习方法研究，探索高分辨率病理图像在弱监督分类任务中的高效表征与聚合方案。
围绕病理切片数据的特征提取、实例组织、模型建模与实验验证展开研究，提升模型对局部病灶区域与全局组织结构信息的综合建模能力。
结合医学场景需求，参与数据处理流程与实验方案设计，为后续模型优化与临床辅助分析提供技术支持。

## 🛠️ 实战项目
### 1. 基于Gin的医院诊疗与信息管理系统的设计与实现
**时间**：2023.09- 2024.01 

项目性质：毕业设计项目 

独立完成医院诊疗与信息管理系统的设计与开发，基于 Gin 搭建后端服务，按照 RESTful API 规范实现用户、诊疗信息与业务数据管理接口。
采用 JWT 实现用户身份认证与接口访问控制，提升系统安全性与会话管理效率。
使用 MySQL 完成业务数据持久化设计，支撑诊疗信息、用户信息及相关业务模块的存储与查询。
基于 Docker 进行项目容器化部署，提升开发、测试与部署环境的一致性，增强系统可维护性与可迁移性。

项目地址：[Github](https://github.com/AlizeLiu/Hospital-Information-System )

### 2.  语音转存与录音管理服务重构
**时间**：2024.03- 2024.05 
项目性质：独立重构项目

负责语音转存与录音管理服务的后端重构，完成 Gin 框架迁移，并将代码重构为 controller、service、storage 分层架构，提升系统可维护性与扩展性。
重构录音元数据接收、查询及仓库信息更新等核心接口，优化对外参数设计与内部业务处理流程。
引入异步写库与异步通知机制，降低主请求链路阻塞风险，提升服务稳定性与处理效率。
完善配置与工程化能力，包括数据库配置拆分、密码加密、自动建表开关控制及多平台打包支持。
配合测试与版本发布完成服务上线，支撑录音数据的稳定接收、存储与管理。

### 3.  告警驱动的租户活动自动关停服务

**时间**：2024.04 – 2024.06
项目性质：实习期间独立需求开发
技术栈：Go、Kafka、Docker、Kubernetes

根据业务侧提出的自动化治理需求，负责告警监听与租户活动关停服务的设计与实现。
基于 Kafka Consumer 实现告警消息消费，完成消息解析、字段提取及触发条件判断。
按照业务规则识别指定异常告警，并联动内部接口执行租户活动自动关停，提高异常响应效率。
负责服务日志、配置管理及部署文件编写，完成容器化部署并推动项目上线。
